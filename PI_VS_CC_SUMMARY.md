# pi-vs-cc — Project Summary

Generated by executing the `/prime` workflow defined in [experimental-learning/agentic-ai/pi-vs-claude-code/.claude/commands/prime.md](experimental-learning/agentic-ai/pi-vs-claude-code/.claude/commands/prime.md).

## Purpose

`pi-vs-cc` (located at [experimental-learning/agentic-ai/pi-vs-claude-code/](experimental-learning/agentic-ai/pi-vs-claude-code/)) is an **extension playground for the Pi coding agent**. It is a curated collection of progressively more ambitious extensions that demonstrate how to customise Pi's TUI, hook into its event lifecycle, register tools/commands/widgets, and orchestrate sub-agents.

The repo doubles as an instructive comparison point against Claude Code (CC) — hence the name — with `COMPARISON.md`, `PI_VS_OPEN_CODE.md`, `RESERVED_KEYS.md`, and `TOOLS.md` capturing the differences.

## Theme Conventions

[THEME.md](experimental-learning/agentic-ai/pi-vs-claude-code/THEME.md) defines a strict colour grammar that every extension follows:

| Token     | Role              | Used For                                      |
| --------- | ----------------- | --------------------------------------------- |
| `success` | Primary value     | Token counts, hash fills, branch name         |
| `accent`  | Secondary value   | Percentages, tool names, token-out counts     |
| `warning` | Punctuation/frame | Brackets, parens, pipes, cost                 |
| `dim`     | Filler/spacing    | Dashes, labels, separators                    |
| `muted`   | Subdued text      | Cwd name, fallback states                     |

A reusable `applyExtensionDefaults()` helper in [extensions/themeMap.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/themeMap.ts) is wired into every extension's `session_start`.

## Extensions (Entry Points)

The [justfile](experimental-learning/agentic-ai/pi-vs-claude-code/justfile) groups them as a curriculum:

### g1 — TUI customisation primitives
1. `pi` — vanilla baseline.
2. [pure-focus.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/pure-focus.ts) — strips footer/status entirely.
3. [minimal.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/minimal.ts) — model name + 10-block context meter footer.
4. [cross-agent.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/cross-agent.ts) — auto-loads `commands/`, `skills/`, `agents/` from sibling `.claude/`, `.gemini/`, `.codex/` dirs (project + global) and registers them as `/cmd`, `/skill:name`, `@agent`.
5. [purpose-gate.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/purpose-gate.ts) — blocks input until the user declares an agent purpose; injects it into the system prompt and pins it as a widget.
6. [tool-counter.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/tool-counter.ts) — rich two-line footer (model + context meter, tokens in/out + cost, cwd + git branch, per-tool tally).
7. [tool-counter-widget.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/tool-counter-widget.ts) — same data but as an above-editor widget with per-tool background colours.
8. [subagent-widget.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/subagent-widget.ts) — `/sub`, `/subcont`, `/subrm`, `/subclear` spawn background Pi sub-agents with persistent JSONL sessions and live-streaming widgets.
9. [tilldone.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/tilldone.ts) — task-driven discipline. Registers a `tilldone` tool and **blocks all other tools until tasks are defined and one is in-progress**; auto-nudges on `agent_end` if work remains. Surfaces in footer, widget, status, and `/tilldone` overlay.

### g2 — Multi-agent orchestration
10. [agent-team.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/agent-team.ts) — strips the primary agent of all codebase tools; only `dispatch_agent` is available. Loads agent personas from `agents/`, `.claude/agents/`, `.pi/agents/`, picks an active team from [.pi/agents/teams.yaml](experimental-learning/agentic-ai/pi-vs-claude-code/.pi/agents/teams.yaml), and renders a live grid dashboard of each agent's status, context %, tool count, and last line.
11. [system-select.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/system-select.ts) — `/system` picker that swaps the system prompt to a chosen agent persona and restricts tools accordingly.
12. [damage-control.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/damage-control.ts) — loads YAML-defined safety rules ([.pi/damage-control-rules.yaml](experimental-learning/agentic-ai/pi-vs-claude-code/.pi/damage-control-rules.yaml)): bash pattern blocks, zero-access paths, read-only paths, no-delete paths.
13. [agent-chain.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/agent-chain.ts) — sequential pipeline orchestrator. Chains live in [.pi/agents/agent-chain.yaml](experimental-learning/agentic-ai/pi-vs-claude-code/.pi/agents/agent-chain.yaml); each step's output becomes `$INPUT` for the next; `$ORIGINAL` is always the user's first prompt.

### g3 — Meta
14. [pi-pi.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/pi-pi.ts) — meta-agent that builds Pi extensions/themes/skills. A team of 9 read-only research experts ([.pi/agents/pi-pi/](experimental-learning/agentic-ai/pi-vs-claude-code/.pi/agents/pi-pi/) — `ext-expert`, `theme-expert`, `skill-expert`, `config-expert`, `tui-expert`, `prompt-expert`, `agent-expert`, `cli-expert`, `keybinding-expert`) run in parallel via a `query_experts` tool; the `pi-orchestrator` agent synthesises and writes files.

### Utilities
15. [session-replay.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/session-replay.ts) — scrollable timeline overlay of session history.
16. [theme-cycler.ts](experimental-learning/agentic-ai/pi-vs-claude-code/extensions/theme-cycler.ts) — `Ctrl+X` / `Ctrl+Q` cycle themes; `/theme` picker; flashes a swatch widget on switch.

The `just open <ext>...` and `just all` recipes spawn each extension stack in its own macOS Terminal window for side-by-side demoing.

## Agent Library ([.pi/agents/](experimental-learning/agentic-ai/pi-vs-claude-code/.pi/agents/))

Each agent is a markdown file with a YAML frontmatter (`name`, `description`, `tools`) followed by its system-prompt body. Tools are restricted per agent.

| Agent          | Role                                              | Tools                          |
| -------------- | ------------------------------------------------- | ------------------------------ |
| `scout`        | Read-only recon and codebase exploration          | read, grep, find, ls           |
| `planner`      | Architecture and step-by-step plans (read-only)   | read, grep, find, ls           |
| `plan-reviewer`| Critiques plans (read-only)                       | read, grep, find, ls           |
| `builder`      | Implements changes                                | read, write, edit, bash, grep, find, ls |
| `reviewer`     | Code review + tests                               | read, bash, grep, find, ls     |
| `red-team`     | Security/adversarial testing                      | read, bash, grep, find, ls     |
| `documenter`   | Writes docs                                       | read, write, edit, grep, find, ls |
| `bowser`       | Headless browser automation via Playwright skill  | (skill-driven)                 |

Teams ([.pi/agents/teams.yaml](experimental-learning/agentic-ai/pi-vs-claude-code/.pi/agents/teams.yaml)): `full`, `plan-build`, `info`, `frontend`, `pi-pi`.

Chains ([.pi/agents/agent-chain.yaml](experimental-learning/agentic-ai/pi-vs-claude-code/.pi/agents/agent-chain.yaml)): `plan-build-review`, `plan-build`, `scout-flow` (triple-scout recon), `plan-review-plan` (plan → critique → revise), `full-review` (scout → plan → build → review).

## Conventions (from [CLAUDE.md](experimental-learning/agentic-ai/pi-vs-claude-code/CLAUDE.md))

- Extensions are standalone `.ts` files loaded by Pi's jiti runtime.
- Register tools/commands/shortcuts at the **top level** of the extension function — not inside event handlers (the `cross-agent` extension explains why: late registration is silently dropped).
- Use `isToolCallEventType()` for type-safe `tool_call` event narrowing.
- TypeBox for tool parameter schemas; `StringEnum` from `pi-ai` for Google-compatible enums.
- Settings ([.pi/settings.json](experimental-learning/agentic-ai/pi-vs-claude-code/.pi/settings.json)) point Pi's prompt loader at `../.claude/commands` so `/prime` resolves cross-tool.
